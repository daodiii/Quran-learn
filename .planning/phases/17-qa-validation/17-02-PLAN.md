# Plan 17-02: Glossary Link Validator + Readability Progression Checker

## Completed

### Link Validator (validate-links.ts) — NEW
- Glossary anchor validation: checks [text](/resources/glossary#anchor) against actual IDs
- Lesson cross-reference validation: checks /learn/level-N/slug paths
- Handles numeric prefix stripping (01-slug → slug for Astro routing)
- Resource link validation with known slug aliases (verb-tables → verb-conjugation-tables, etc.)
- Scans both lesson and resource MDX files
- Reports broken links as warnings (content issues needing manual fix)

### Progression Checker (validate-progression.ts) — NEW
- Transliteration density by level (L1 ≥75%, L2 ≥60%, L3 ≥30%, L4-5 minimal)
- Exercise count per lesson (minimum 3, flag >6)
- Lesson length by level (generous ranges, flag outliers)
- ArabicExample count (minimum 2 per lesson)
- Section structure check (presence of exercises)

### Integration
- Added validate:links and validate:progression npm scripts
- Link validation integrated into main validate-content.ts orchestrator

## Results
- Link validator found ~600 broken cross-references (mostly slug mismatches between content and filenames)
- Progression checker found only 3 warnings (3 slightly-long lessons)
- All issues are non-blocking warnings appropriate for manual review
